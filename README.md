# 女友工坊 (Memory Workshop)

一个用于构建虚拟女友人格档案的 Web 应用。通过对话式交互，帮助用户创建完整、连贯的虚拟人格记忆，最终生成 `soul.md` 文件。

## 功能特点

- AI 引导式提问，帮助挖掘人格细节
- 支持多种记忆分类管理
- AI 辅助生成和润色记忆内容
- 记忆一致性检查
- 人生时间线可视化
- 对话风格示例生成
- 支持多 LLM 提供商（MiniMax、OpenAI、Anthropic）
- 中英文界面切换

## 安装

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置 API Key

编辑 `.env` 文件，填入你的 API Key：

```env
# MiniMax API (默认)
MINIMAX_API_KEY=your_minimax_api_key_here
MINIMAX_GROUP_ID=your_group_id_here

# OpenAI API (可选)
OPENAI_API_KEY=your_openai_api_key_here

# Anthropic API (可选)
ANTHROPIC_API_KEY=your_anthropic_api_key_here
```

### 3. 启动应用

```bash
python girl.py
```

应用将在 http://localhost:4911 启动。

## 操作流程

### 第一步：填写基本设定

1. 点击左侧菜单「基本设定」
2. 点击右上角「编辑」按钮
3. 填写模板中的各项信息（姓名、生日、外貌、性格等）
4. 点击「保存」

**提示**：基本设定是所有其他记忆的基础，建议优先完成。

### 第二步：建立社会关系

1. 点击左侧菜单「社会关系」
2. 有两种方式添加内容：

**方式一：AI 辅助生成**
- 在下方输入框输入关键词，如："父亲，中学老师，严厉但关心女儿"
- 点击「生成记忆」按钮
- AI 会生成一段关于父亲的详细描述
- 确认无误后，点击「保存到分类」

**方式二：手动编辑**
- 点击「编辑」按钮
- 按以下格式添加：

```markdown
## 父亲
- 姓名: 于建国
- 年龄: 52
- 职业: 中学教师
- 性格: 严肃但内心温柔
- 与她的关系: 小时候管教严格，长大后成为可以谈心的朋友
- 关键记忆: 高考前夜，父亲第一次说"不管结果如何，爸爸都为你骄傲"
```

### 第三步：使用 AI 提问功能

1. 在控制面板的下拉菜单中选择要填充的分类
2. 点击「AI 提问」按钮
3. AI 会根据当前分类和已有内容，提出一个深入的问题
4. 在下方输入框回答问题
5. 点击「润色」让 AI 扩展你的回答（可选）
6. 点击「保存到分类」将内容保存

**示例流程**：
```
AI 提问: "她小时候最喜欢的玩具是什么？有什么特别的回忆吗？"

你的回答: "一只旧泰迪熊，是外婆送的，睡觉必须抱着"

点击「润色」后 AI 扩展为:
"那只泰迪熊是我五岁生日时外婆送的，棕色的毛已经被我摸得有些发白了。
每天晚上我都要抱着它才能入睡，妈妈说我小时候有一次把它忘在幼儿园，
哭了整整一个晚上..."
```

### 第四步：添加特殊经历

特殊经历需要按时间格式记录，以便生成时间线：

```markdown
## 2004-09 开始学钢琴
妈妈带我去琴行，第一次摸到黑白键，觉得特别神奇...

## 2010-06 小升初考试
那是我第一次感受到考试的压力...

## 2016-06 高考
高考前一晚几乎没睡，但奇怪的是进考场后反而很平静...
```

**格式要求**：`## YYYY-MM 标题`

### 第五步：丰富其他分类

按照同样的方式，逐步填充：

| 分类 | 内容建议 |
|------|----------|
| 日常琐事 | 作息时间、工作内容、通勤方式、午餐习惯等 |
| 事件记忆 | 与男友的第一次见面、约会、旅行、争吵、和好等 |
| 偏好习惯 | 喜欢的食物/颜色/音乐、讨厌的事物、口头禅、小动作 |
| 价值观 | 对爱情/友情/事业的看法，人生信条，处世态度 |
| 情感模式 | 什么让她开心/难过/生气，如何表达情绪，安慰方式 |

### 第六步：检查一致性

当记忆积累到一定程度后：

1. 点击左侧菜单「一致性检查」
2. 点击「开始检查」按钮
3. AI 会扫描所有记忆文件，找出矛盾之处
4. 根据 AI 的建议修正不一致的内容

### 第七步：生成对话风格

1. 点击「对话风格」按钮
2. AI 会根据已有人设，生成不同情绪下的对话示例
3. 可以保存到「情感模式」分类中

### 第八步：导出 Soul 文件

1. 点击右上角「导出 soul.md」按钮
2. 所有分类的内容会合并成一个完整的 `soul.md` 文件
3. 浏览器会触发下载，保存为 `soul.md`

## 记忆文件结构

```
memories/
├── basic.md              # 基本设定
├── social_relations.md   # 社会关系
├── special_experience.md # 特殊经历
├── daily_chores.md       # 日常琐事
├── events.md             # 事件记忆
├── preferences.md        # 偏好习惯
├── values.md             # 价值观
└── emotions.md           # 情感模式
```

（点击「导出 soul.md」时由浏览器下载合并后的 soul.md，不再写入本目录。）

## 设置说明

点击右上角「设置」按钮可以：

- 切换 LLM 提供商（MiniMax / OpenAI / Anthropic / Ollama / 自定义）
- 测试 API 连接

**注意**：API Key 及以下配置均在 `.env` 中填写，不能在界面中修改。

- **Ollama**：`OLLAMA_BASE_URL`（如 `http://127.0.0.1:11434`）、`OLLAMA_MODEL`（如 `llama3.2`）
- **自定义大模型**（OpenAI 兼容接口）：`CUSTOM_BASEURL`、`CUSTOM_APIKEY`、`CUSTOM_MODEL`

## 使用技巧

1. **循序渐进**：先完成基本设定和社会关系，再填充其他内容
2. **保持一致**：注意人物关系、时间线的一致性
3. **善用 AI**：不确定如何描述时，让 AI 生成初稿再修改
4. **定期检查**：使用一致性检查功能避免矛盾
5. **备份文件**：重要修改前备份 `memories` 文件夹

## 常见问题

**Q: API 调用失败怎么办？**
A: 检查 `.env` 文件中的 API Key 是否正确，确认账户余额充足。

**Q: 如何切换语言？**
A: 点击右上角的语言切换按钮（中文/EN）。

**Q: 生成的内容不满意怎么办？**
A: 可以多次点击生成按钮获取不同版本，或手动编辑修改。

**Q: 时间线不显示怎么办？**
A: 确保「特殊经历」中的事件使用了正确的格式：`## YYYY-MM 标题`
